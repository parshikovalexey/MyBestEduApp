# Промпт для исследования VK по монетизации и оплатам

Скопируйте этот промпт в Deepseek (или аналогичный AI):

---

**РОЛЬ:** Эксперт по разработке VK Mini Apps и платёжным системам ВКонтакте

**ЗАДАЧА:** Исследовать документацию ВКонтакте по теме монетизации и приёма платежей для образовательного продукта (курс по созданию игр для детей).

**КОНТЕКСТ ПРОЕКТА:**
- VK Mini App для детей 10-16 лет
- Продажа образовательного курса (~3000-5000₽)
- Интеграция в экосистему ВКонтакте

**ВОПРОСЫ ДЛЯ ИССЛЕДОВАНИЯ:**

1. **VK Payments в Mini Apps:**
   - Можно ли принимать оплату напрямую через VK для физических товаров/услуг?
   - Какие есть варианты приёма платежей?
   - Какие комиссии и ограничения?

2. **Внешние платёжные системы:**
   - Можно ли использовать внешние платёжные системы (ЮKassa, Сбер, Тинькофф) внутри VK Mini App?
   - Есть ли ограничения от ВК на редиректы на внешние сайты оплаты?
   - Как это работает с точки зрения пользователя?

3. **VK Donut / Подписки:**
   - Можно ли продавать курс через VK Donut?
   - Есть ли механика разовых покупок (не подписок)?

4. **VK Ads для монетизации:**
   - Как встроить рекламу в игры (interstitial, rewarded)?
   - Какой доход ожидать (~30₽/100 показов — это реально?)

5. **Юридические аспекты:**
   - Что нужно для приёма платежей (ИП/ООО)?
   - Есть ли упрощённые схемы для ВК-разработчиков?

6. **Практические кейсы:**
   - Кто уже продаёт курсы/образование через VK Mini Apps?
   - Как они это делают (ссылки, примеры)?

**ФОРМАТ ОТВЕТА:**
- Структурированный ответ по каждому пункту
- Ссылки на официальную документацию VK
- Практические рекомендации для нашего MVP
- Оценка сложности реализации (просто/средне/сложно)

**ЦЕННОСТЬ ОТВЕТА:**
На основе исследования нужно понять — строить ли оплату внутри ВК или делать внешний сайт-лендинг с оплатой и редирект в VK Mini App для доступа к контенту.
