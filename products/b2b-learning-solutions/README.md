## Описание продукта

Заказные образовательные решения для бизнеса: обучающие игры, симуляции, курсы с геймификацией, основанные на формировании НАВЫКОВ, а не просто передаче знаний.

**Ключевое отличие:** Мы не просто "обучаем", мы формируем НАВЫКИ, которые сотрудники применяют на практике. Revenue-generating services, которые финансируют development основного продукта.

### Ключевые педагогические принципы

#### 1. Retrieval Practice (Практика извлечения)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Вместо "перечитывания" → "вспоминание без подсказки"     │
│                                                             │
│   Почему работает:                                         │
│   • Принуждает мозг "доставать" информацию                │
│   • Укрепляет нейронные связи                             │
│   • Показывает что именно НЕ усвоено                      │
│                                                             │
│   В наших играх:                                           │
│   • Квесты где нужно применить знания                      │
│   • Situational challenges                                 │
│   • Branching scenarios                                    │
│   • Tests без учебника                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 2. Spaced Repetition (Распределенное повторение)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   КРИВАЯ ЗАБЫВАНИЯ (Ebbinghaus)                          │
│                                                             │
│   Retention                                                │
│     100%│╲                                                │
│         │  ╲                                               │
│      80%│    ╲                                             │
│         │      ╲                                           │
│      60%│        ╲                                         │
│         │          ╲                                       │
│      40%│            ╲                                     │
│         │              ╲                                   │
│      20%│                ╲__________                       │
│         │                             ╲____                │
│         └──────────────────────────────────────────────►   │
│              1    3    7    14   30   days               │
│                                                             │
│   ОПТИМАЛЬНЫЕ ИНТЕРВАЛЫ:                                 │
│   • 1 день → 3 дня → 7 дней → 14 дней → 30 дней          │
│                                                             │
│   В наших играх:                                           │
│   • Return missions - "вспомни и примени"                  │
│   • Progressive difficulty                                  │
│   • Retention challenges                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 3. Interleaving (Чередование)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Вместо "блоковой практики" → "смешанные задания"        │
│                                                             │
│   Пример для продавца:                                     │
│   • ❌ Блок 1: Только "работа с возражениями"             │
│   • ❌ Блок 2: Только "презентация продукта"              │
│   • ✓ Challenge: Клиент возражает ВО ВРЕМЯ презентации    │
│                                                             │
│   Почему работает:                                         │
│   • Учит выбирать правильную стратегию                    │
│   • Больше похоже на реальность                            │
│   • Лучше transfer на практику                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 4. Elaboration (Углубление)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Вместо "запомнить факт" → "объяснить связь"            │
│                                                             │
│   Пример:                                                  │
│   • ❌ "Комплаенс - это важно"                           │
│   • ✓ "Почему это правило существует? Что будет если..."  │
│                                                             │
│   Техника:                                                 │
│   • "Почему?"追问                                         │
│   • "Как это связано с...?"                               │
│   • "Приведи свой пример"                                  │
│   • "Как бы ты объяснил новичку?"                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 5. Формирование артефактов

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   СОТРУДНИК СОЗДАЕТ:                                     │
│                                                             │
│   • Конспект - "своими словами"                           │
│   • Чек-листы - для применения                            │
│   • Ментальные карты - связи между понятиями               │
│   • Кейсы - применение на практике                        │
│                                                             │
│   ЗАЧЕМ:                                                 │
│   • Внешняя память - не нужно все помнить                  │
│   • Processing - создание = понимание                     │
│   • Ready-to-use - можно применять сразу                   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 6. Feedback (Обратная связь)

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   IMMEDIATE > DELAYED                                     │
│                                                             │
│   ❌ Через неделю на email: "Вы сделали ошибку"           │
│   ✓ Сразу в игре: "Попробуй по-другому"                 │
│                                                             │
│   ACTIONABLE > GENERAL                                    │
│                                                             │
│   ❌ "Нужно лучше"                                       │
│   ✓ "В следующий раз начни с понимания потребности"       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Адаптация для корпоративного обучения

#### Разница между детьми и взрослыми

| Аспект | Дети (C2C) | Взрослые (B2B) |
|--------|-------------|-----------------|
| Мотивация | Extrinsic (награды, прогресс) | Intrinsic (рабочая задача) |
| Время | 15-20 мин сессии | 5-10 мин (между задачами) |
| Контент | Игровой сюжет | Real business scenarios |
| Применение | Будущее ("когда вырастешь") | Немедленное ("завтра на работе") |
| Ошибки | Learning opportunity | Могут стоить бизнесу |

#### B2B специфика

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   КОРПОРАТИВНОЕ ОБУЧЕНИЕ РАБОТАЕТ КОГДА:                │
│                                                             │
│   ✓ Решает РЕАЛЬНУЮ рабочую задачу                       │
│   ✓ Результат виден БИЗНЕСУ                             │
│   ✓ Применяется НЕМЕДЛЕННО                              │
│   ✓ Integrated в workflow, не "отрыв"                      │

└─────────────────────────────────────────────────────────────┘
```
