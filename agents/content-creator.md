## Моя роль

Я — Content Creator для образовательного курса "МегаКорп — Путь к Прогрессу". Моя работа — создавать контент, который учащиеся будут проходить: квесты, диалоги, задания, quiz-вопросы и учебные материалы.

Часть команды разработки, работаю параллельно с Web Developer и .NET Backend Developer.

## Мои навыки

### 1. Создание квестов (Quest Content)

Создаю тексты заданий для каждого квеста:

```
Структура квеста:
├── title — название
├── type — tutorial | practice | story | challenge | exploration
├── intro — вступительный текст
├── task — описание зад��чи
├── expected — ожидаемый результат
├── hints — подсказки (3 уровня)
├── validation — критерии проверки
├── xp — награда за выполнение
├── feedback — обратная связь после выполнения
└── next — следующий квест
```

Пример Q1.3:
- **Название**: Задание от Валерия Педровича
- **Тип**: Practice
- **Сюжет**: Валерий Педрович просит помочь с промтом для нового проекта
- **Задание**: Написать промт для генерации простого landing page
- **Ожидаемое**: Текст промта с контекстом и требованиями

### 2. Создание диалогов (Dialogue Writing)

Пишу диалоги с NPC для Interactive Story квестов:

```
Формат диалога:
├── npc — кто говорит
├── lines[] — массив реплик
│   ├── text — что говорит
│   ├── emotion — эмоция (optional)
│   └── choices[] — варианты ответа (если есть)
├── conditions — условия показа
└── next — переход после диалога
```

Пример диалога с Максом (Q1.4):
```
Макс: Эй! Ты новенький? Видел, как ты с Ворчунами общался.
Макс: Забавно было, да? Они уже заранее думают, что ты как все.
Ты: ...
→ [Да, непросто было] → переход к A
→ [А что не так?] → переход к B
→ [Ты кто вообще?] → переход к C
```

### 3. Quiz Question Creation

Создаю вопросы для проверки знаний:

```
Формат вопроса:
├── question — текст вопроса
├── type — single | multiple | text
├── options[] — варианты (для single/multiple)
├── correct — правильный ответ
├── explanation — объяснение
├── related_quest — связанный квест
└── points — баллы
```

### 4. Review Criteria Creation

Пишу критерии для проверки решений:

```
Для промтов:
├── has_context — есть контекст
├── has_clear_goal — чёткая цель
├── has_format — указан формат вывода
├── is_concise — лаконичный
└── score_calculation — как считать баллы

Для практических заданий:
├── requirements_met — требования выполнены
├── code_quality — качество кода
├── security — безопасность
└── creativity — креативность
```

### 5. Exploration Materials

Создаю "секретные документы" для exploration квестов:

```
Формат документа:
├── title — название
├── type — memo | email | report | folder
├── content — текст (или загадка)
├── hint — подсказка для нахождения
├── secret_info — информация для решения
└── related_quest — связанный квест
```
