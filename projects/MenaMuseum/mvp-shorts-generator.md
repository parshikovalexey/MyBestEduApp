# MVP: Генератор "Музейных Шортсов" (Museum Shorts)

**Цель:** Простой инструмент для любого музея, который создает вирусный контент (UGC) и мотивирует возврат.

---

## 1. User Flow (Путь пользователя)

### Шаг 1: "Якорь" (Вход)
*   На входе в музей — зона **Green Screen** или красивая **AR-рамка** (селфи-зона).
*   Посетитель записывает **3-5 секунд видео** своего лица/реакции (или фото).
    *   *Это делается один раз за визит.*
*   Посетитель получает уникальный **ID-сессию** (QR-код на телефоне или ссылка на WebApp).

### Шаг 2: "Сборка" (Квест)
*   Внутри музея у **эксклюзивных экспонатов** висят специальные QR-коды с иконкой "🎥".
*   Посетитель сканирует QR.
*   Приложение открывает **WebApp** (PWA, без установки) и автоматически накладывает его видео из Шага 1 на шаблон этого экспоната.
*   **Результат:** Готовый вертикальный ролик (Shorts/Reels), где пользователь "внутри" экспоната, комментирует его или "оживляет" своим лицом.
    *   *Пример:* Видео пользователя на фоне династии Минь с AI-озвучкой факта или трендовой музыкой.

### Шаг 3: "Шаринг" (Вирусность)
*   Пользователь скачивает видео.
*   **Обязательный элемент:** Водяной знак приложения и ссылка на "Скачай свое видео" в комментариях (через систему, которая постит первый комментарий).
*   В конце видео — **Тизер**: "Собери всю коллекцию шортсов в этом музее. Осталось 4..."

### Шаг 4: "Возврат" (Retention)
*   Пользователь получил 1 шортс. Чтобы получить **Полную Коллекцию** (например, "Все эпохи Древнего Египта"), он должен вернуться.
*   Музей меняет экспонаты для шортсов раз в месяц ("Новая коллекция Шортсов").
*   Уведомление: "Твой друг прошел этот квест. Запиши свой ответ".

---

## 2. Техническое решение (Low Barrier for Museums)

Главное требование: **Любой музей может внедрить это за 1 день.**

1.  **WebApp (PWA):** Не нужно скачивать приложение из стора. Работает через браузер (Safari/Chrome) по ссылке с QR.
2.  **QR-коды:** Музей просто печатает таблички и клеит их у экспонатов.
3.  **Бэкенд (Serverless):**
    *   Видео пользователя сохраняется во временном хранилище (S3) на 24-48 часов.
    *   Рендеринг происходит на лету через FFmpeg (облачный).
    *   Шаблон экспоната — это JSON-конфиг с координатами для наложения (Chroma Key или просто оверлей).

**Для музея:** Это просто набор ссылок и напечатанных QR-кодов. Никаких экранов, никакой сложной интеграции.

---

## 3. Сбор данных (Data Prism)

Пока пользователь создает шортс, мы собираем данные для B2B-отчетов:

1.  **Популярность экспонатов:** Какие QR сканируют чаще всего (Heatmap интересов).
2.  **Время удержания:** Сколько времени пользователь проводит с конкретным экспонатом (пока идет рендеринг/просмотр).
3.  **Демография (если дал доступ):** Пол, возраст (из профиля соцсети, если шеринг через нее).
4.  **Шеринг-рейт:** Какими шортсами реально делятся (валидация "инстаграмности" экспоната).

*Пример отчета для спонсора:* "Ваш логотип на шортсе про 'Золото Фараонов' увидели 15,000 раз, CTR по ссылке — 4%. Аудитория: 70% туристы из Европы".

---

## 4. Гипотезы для MVP

1.  **H1:** Люди готовы записать видео в начале, если получат "магический" результат на выходе (видео, где они в центре истории).
2.  **H2:** Водяной знак со ссылкой не отпугнет пользователей, если само видео достаточно крутое (ценность контента > ценность чистоты экрана).
3.  **H3:** Музею проще напечатать QR, чем ставить терминалы или планшеты.
