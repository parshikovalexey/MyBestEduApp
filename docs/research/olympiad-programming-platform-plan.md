## 1. Фокус и стратегия

**Направление:** исключительно олимпиадное программирование (алгоритмы, структуры данных, competitive programming).

**Миссия:** Создать системную среду, где каждый ребёнок соревнуется не с другими, а со своей вчерашней версией. Индивидуальный трек роста, прозрачные метрики прогресса, путь от первого «Hello World» до регионального этапа ВсОШ.

**Стратегическая цель:** Через 2–3 года собрать команду, которая отбирается на региональный этап ВсОШ по информатике (программирование).

**Философия:** «Становимся лучше каждый день» — персональный прогресс-трек вместо рейтинга между учениками.

| Курс | Зачем |
|------|-------|
| Введение в программирование на Python | Базовый синтаксис, достаточен для старта |
| Введение в алгоритмы: Python/C++ | Алгоритмы сортировки, поиска — основа олимпиад |

**Для старшей группы (8–10 классы):**
| Курс | Зачем |
|------|-------|
| Введение в алгоритмы: реализация на Python | Алгоритмы, структуры данных |
| Введение в алгоритмы: реализация на C++ | C++ — основной язык олимпиад |
| Дополнительные главы алгебры | Математическая база для алгоритмов |
| Теория вероятностей | needed для продвинутых задач |

### Чего НЕ хватает в Сириус.Курсах для нашей задачи:

- **Нет комплексной многолетней программы** — курсы разрознены, нет связного пути от 5 до 11 класса
- **Нет системы персонального трека** — нет «карты роста» ученика с метриками
- **Нет интеграции с олимпиадным календарём** — курсы не привязаны к этапам ВсОШ
- **Нет командной работы** — всё индивидуально, нет групповых контестов
- **Нет системы наставничества** — нет связи «тренер → ученик → родитель»
- **Нет постепенного перехода** от Python к C++ (олимпиадный стандарт)

**Вывод:** Сириус.Курсы — отличный дополнительный ресурс (видеолекции, задачи с разборами), но ядро обучения строим сами с нуля.

## 4. Каркас платформы

### 4.1 Что БУДЕТ на платформе

#### Роли пользователей:
- **Ученик** — основная роль, доступ к заданиям, контестам, своему треку
- **Тренер** — создание контестов, назначение задач, просмотр аналитики, комментарии
- **Родитель** — просмотр прогресса ребёнка, посещаемости, уведомлений
- **Администратор** — управление пользователями, группами, расписанием

#### Ядро платформы:

**1. Индивидуальный план развития (ИПР)**
- Карта тем и навыков для каждого уровня (младший/старший/продвинутый)
- Визуальный «дерево навыков» — какие темы пройдены, какие в процессе
- Автоматическая рекомендация: «тебе пора переходить к графам»
- Персональные цели на месяц/квартал/год

**2. Персональный прогресс-трек**
- Количество решённых задач (общее / по темам / по сложности)
- История контестов: дата, задачи, результат, время решения
- График роста: динамика решаемых задач по неделям
- «Сегодня ты лучше, чем вчера» — сравнение только с собой
- Достижения/бейджи: «10 задач решено», «Первый контест», «Решил задачу D»

**3. Задачник**
- База задач с классификацией: тема, сложность, источник
- Интеграция с внешними судьями: Codeforces API, Яндекс.Контест
- Собственные задачи тренеров
- Видеоразборы и текстовые решения (открываются после попытки)
- Тегирование: «ВсОШ 2024 муниципальный», «Графы — DFS»

**4. Контесты**
- Создание контестов тренером: выбор задач, таймер, видимость
- Режим «турнир» — leaderboard, но только после завершения
- Режим «тренировка» — без таймера, можно смотреть подсказки
- Разбор после контеста: статистика, кто какую задачу решил
- Архив всех контестов

**5. Система домашних заданий**
- Тренер назначает задачи на неделю
- Дедлайн, напоминания
- Статус: не начато / в процессе / решено / требует проверки
- Комментарии тренера к решению

**6. Аналитика для тренера**
- Дашборд группы: кто как продвигается
- Выявление отстающих тем: «80% группы не освоили бинарный поиск»
- Рекомендации: «Иванову пора на старшую группу»
- Экспорт отчётов для НКО и родителей

**7. Календарь олимпиад**
- Автоматическое добавление дат ВсОШ и других олимпиад
- Напоминания о регистрации
- Трекер участия: на каких олимпиадах был, результат
- Целевая олимпиада: «Следующая цель — муниципальный этап 2 декабря»

**8. Коммуникации**
- Объявления тренера группе
- Уведомления родителям: «Решено 15 задач на этой неделе»
- Канал связи тренер ↔ родитель (без чата, только structured messages)

#### Интеграции:
- **Codeforces API** — импорт задач, рейтинг (для старших)
- **Яндекс.Контест** — проведение собственных контестов
- **Сириус.Курсы** (edu.sirius.online) — ссылки на релевантные модули в ИПР
- **Telegram Bot** — уведомления, напоминания

### 4.2 Чего НЕ БУДЕТ на платформе (MVP + 2 года)

- **Социальная сеть / чат между учениками** — фокус на учёбе, не на общении
- **Геймификация-лидерборд между учениками** — принципиально: соревнуемся с собой, не друг с другом
- **AI-автопроверка решений** — на старте проверка руками тренера (точнее, и надёжнее)
- **Видеозвонки / стриминг занятий** — используем Zoom/Telegram, не строим свой
- **Маркетплейс курсов** — только наша программа
- **Мобильное приложение** — веб-адаптив достаточно (на занятиях — компьютеры)
- **Оплата внутри платформы** — пожертвования через сайт НКО отдельно

### 4.3 Архитектура (предварительно)

```
Frontend:  React/Vue.js (SPA), адаптивный дизайн
Backend:   .NET 8 API (учитывая имеющегося разработчика)
Database:  PostgreSQL
Auth:      JWT + роли (ученик/тренер/родитель/админ)
External:  Codeforces API, Яндекс.Контест API, Telegram Bot API
Hosting:   российский хостинг (Timeweb / Selectel / Yandex Cloud)
```

### 4.4 Приоритизация: что делаем летом (MVP к сентябрю)

| Спринт | Дата | Что делаем |
|--------|------|-----------|
| MVP-1 | Июнь | Auth, роли, профиль ученика, базовый задачник |
| MVP-2 | Июль | ИПР (карта тем), назначение задач тренером, прогресс-трек |
| MVP-3 | Август | Контесты, аналитика тренера, интеграция с Codeforces |
| MVP-4 | Август | Календарь олимпиад, уведомления, родительский дашборд |

### 5.4 Подход «Алгоритм прежде кода» для младших групп

**Гипотеза:** Для детей 5–7 классов приоритетом должно быть развитие алгоритмического мышления, а не изучение синтаксиса программирования. Написание алгоритма (логики решения) важнее, чем его кодирование на конкретном языке.

**Обоснование релевантности:**
1. **Синтаксический барьер:** Начинающие тратят до 70% умственных усилий на борьбу с синтаксисом (скобки, точки с запятой, отступы), а не на саму задачу
2. **Когнитивная нагрузка:** Алгоритмическое мышление (структурирование решения) и синтаксис (запись решения) — разные навыки, которые лучше развивать последовательно
3. **Международная практика:** В странах-лидерах по олимпиадному программированию (Сингапур, Южная Корея, Эстония) используется блочное программирование как первый этап
4. **Эффективность перехода:** Дети, начинающие с визуального программирования, быстрее осваивают текстовые языки позже благодаря уже сформированному алгоритмическому мышлению

**Инструменты для реализации:**
- **Blockly** (https://developers.google.com/blockly) — библиотека Google для создания визуальных программируемых блоков, генерирует код на Python, JS, PHP, Dart, Lua
- **Чертёжник-Blockly** (https://kpolyakov.spb.ru/school/blockly/drw-blockly.htm) — адаптация Blockly под задачи алгоритмизации (из системы «Исполнители»)
- **Водолей-Blockly** (https://kpolyakov.spb.ru/school/blockly/vod-blockly.htm) — версия для линейных алгоритмов, циклов, ветвлений
- **mBlock** (https://mblock.cc/) — платформа с плавным переходом от блоков к Python/C++
- **BlockPy** (http://blockpy.ru/) — среда с двухсторонней конвертацией блоков ↔ Python
- **Turtle graphics** через блоки — визуализация алгоритмов через движение черепахи

**Поэтапное внедрение в программу младшей группы:**

**Месяцы 1–3 (Сентябрь–Ноябрь):**
- Исключительно визуальное программирование в блоковых средах
- Задачи на построение алгоритмов: линейные алгоритмы, ветвления, простые циклы
- Акцент на объяснение логики решения: «Как бы ты это сделал поэтапно?»
- Генерация кода из блоков (Python) для демонстрации связи

**Месяцы 4–6 (Декабрь–Февраль):**
- Смешанный режим: 70% блоки + 30% простой Python (без сложного синтаксиса)
- Задачи: циклы с счётчиком, работа со списками, простые функции
- Первые попытки писать код с подсказками-шаблонами

**Месяцы 7–9 (Март–Май):**
- Переход к 50/50: блоки для сложных алгоритмов, Python для закрепления
- Задачи: вложенные циклы, строковые операции, рекурсия (визуализация)
- Акцент на чтение и модификацию чужого кода (сгенерированного из блоков)

**Год 2+:**
- Постепенное сокращение использования блоков до 20–30% времени
- Блоки используются только для планирования сложных алгоритмов перед кодированием
- К концу года 2 младшая группа должна быть готова к полному переход на текстовое программирование

**Интеграция с платформой:**
- В задачнике платформы будет возможность выбора режима: блоки / Python / оба
- При решении в блоковой среде — автоматическая генерация Python-кода для контестера
- Возможность сохранять и делиться алгоритмами как схемами блоков
- Трекер прогресса: % времени в блоковом vs текстовым режиме

## 6. Социальное НКО: план

### 6.1 Почему статус социального НКО важен

- Налоговые льготы (освобождение от НДС для образовательных услуг)
- Доступ к грантам (Президентский фонд культурных инициатив, Фонд поддержки детей)
- Возможность получать целевые пожертвования с налоговым вычетом для доноров

- Методическую поддержку учителей

### 7.2 Зачем нам партнёрство

- **Легитимность:** ассоциация с федеральной структурой
- **Доступ к ресурсам:** методические материалы, курсы, площадки
- **Отбор на смены Сириус:** прямая рекомендация наших лучших учеников
- **Методическая поддержка:** доступ к программам ЦПМК
- **Возможность стать региональной площадкой:** проводить олимпиады у нас

## 8. Финансирование и пожертвования

### 8.1 Модель финансирования

| Источник | Доля (цель) | Комментарий |
|----------|-------------|-------------|
| Пожертвования (физлица) | 30% | Через сайт НКО, платформы |
| Гранты (фонды) | 40% | ПФКИ, Фонд президентских грантов |
| Корпоративные спонсоры | 20% | IT-компании региона |
| Региональные субсидии | 10% | Поддержка одарённых детей |

### 8.2 Каналы сбора пожертвований
 
1. **Сайт НКО** — страница «Поддержать проект» с формой оплаты
2. **Донорская стена** — интерактивная страница с ячейками для логотипов/имен, каждая ячейка покупается за пожертвование (чем больше ячеек — больше место)
3. **Платформа «Планета.ру»** — краудфандинг для социальных проектов
4. **Добро.Mail.ru** — агрегатор благотворительных сборов
5. **СМС-донаты** — через оператора (код 3443, сумма на выбор)
6. **Корпоративные партнёрства** — спонсорство групп, контестов
7. **Регулярные подписки** — «Стань наставником» (ежемесячное пожертвование)

### 8.3 Аргументы для доноров

- **«Каждый ребёнок — это будущий инженер/учёный»** — социальный лифт через образование
- **Прозрачность** — отчёты о расходовании средств на сайте
- **Измеримый результат** — «500 ₽ = 1 час занятия для одного ребёнка»
- **Признание** — имя донора на сайте, на мерче кружка
- **Налоговый вычет** — для юридических лиц (до 1% выручки)

### 8.4 Бюджет: минимальный запуск (на 32 ученика)

| Статья расходов | В месяц | Комментарий |
|-----------------|---------|-------------|
| Аренда помещения | 30 000–60 000 ₽ | Зависит от региона, можно получить бесплатно от муниципалитета |
| Зарплата тренеров (2 чел.) | 80 000–120 000 ₽ | 2 тренера × 6 занятий/неделю |
| Платформа (хостинг, домен) | 5 000 ₽ | Timeweb / Selectel |

| Интернет, расходники | 5 000 ₽ | |
| **Итого** | **120 000–190 000 ₽/мес.** | |

**Стоимость 1 ученика:** ~4 000–6 000 ₽/мес. (6 занятий)
